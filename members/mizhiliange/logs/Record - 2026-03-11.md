### Record - 2026-03-11

### What I did
Today I tried to implement and containerize a custom RCA algorithm.

- Write an algorithm using the Algorithm interface
- Test it locally
- Package it as a Docker container
- Run it locally with `container run`

### What I learned
Implement the Algorithm base class
Use AlgorithmArgs correctly
Return ranked AlgorithmAnswer objects
Handle traces, logs, and fault windows
eval single is flexible and forgiving about input schema
container run is **strictly protocol-driven** and assumes official rcabench datasets

### Problems
Version mismatch between rcabench-platform v2 and v3
Strict requirements for `injection.json`
Datapack naming rules
Read-only volume issues

### What I learned from problems
I tried to fix these by manually mocking Parquet files, but the errors remained
The container pipeline needs official rcabench datasets, not hand-written mock data.


For local work:
Use eval single to iterate on algorithms
Build the Docker image
Leave container run to the real platform with real data
